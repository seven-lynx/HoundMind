# v2025.11.01 — Desktop Simulation, Lite Face Backend, and Setup Docs
> Author: 7Lynx · Doc Version: 2025.11.01

Release tag: `v2025.11.01`

## Highlights
- Desktop PiDog simulation shim (env-driven)
- New Lite Face backend (no dlib), tuned for Raspberry Pi 3B
- Face recognition setup guide and streamlined lightweight install
- Training CLI and smoke test for the lite backend

## Details

### Simulation
- `pidog/__init__.py`: safe no-op motion and plausible sensors when `HOUNDMIND_SIM=1`
- Use `HOUNDMIND_SIM_RANDOM=1` for light sensor noise while testing

### Face (Lite) — Pi 3B friendly
- `packmind/services/face_recognition_lite_service.py`
  - Haar cascade face detection; low-res grayscale frames
  - Optional identity via LBPH when `opencv-contrib-python` is available
  - Detection-only mode works with plain OpenCV (no contrib)
- Config:
  - `FACE_BACKEND = "lite"` selects the lightweight backend
  - `FACE_LITE_DATA_DIR` sets the training asset path (default: `data/faces_lite`)
  - Pi 3B preset defaults to `lite` if face recognition is enabled
- Utilities:
  - `scripts/train_faces_lite.py` — train LBPH from `data/faces_lite/<name>/*.jpg`
  - `tools/lite_face_smoke_test.py` — run lite backend for N seconds and log detections

### Docs
- `docs/face_recognition_setup.md` — Raspberry Pi guidance: piwheels, build tools, swap, disable option, and the lite backend path
- `README.md` / `packmind/README.md` — usage notes for lite backend, CLI, and smoke test
- `docs/packmind_config_guide.md` — added `FACE_BACKEND` and `FACE_LITE_DATA_DIR`

## Install notes (Pi 3B)
- Detection-only: `opencv-python` is sufficient
- Identity: install `opencv-contrib-python` for LBPH (`cv2.face`)
