## Added
- `pidog_sim` package: `from pidog_sim import Pidog` always uses the simulator
- `tools/run_telemetry.py`: Runner that starts the telemetry server using config defaults; supports `--host`, `--port`, `--force`
- `docs/telemetry_quickstart.md`: Install and usage for the telemetry dashboard (P1)

## Changed
- `pidog` shim hardened to avoid self-import and support multiple vendor class name variants (`Pidog`, `PiDog`, etc.). Uses hardware on the Pi when available; otherwise falls back to sim.
- `packmind/orchestrator.py` direct-run support with repo-root bootstrap
- `packmind/__main__.py` so `python -m packmind` launches the orchestrator
- Telemetry flags consolidated in `packmind/packmind_config.py` (`TELEMETRY_ENABLED`, `TELEMETRY_HOST`, `TELEMETRY_PORT`, `TELEMETRY_BASIC_AUTH` placeholder). Removed duplicate `TELEMETRY_ENABLED` under logging.
- README: clarified pidog vs pidog_sim, added `python -m packmind`, and telemetry quickstart link
- Docs: normalized voice instructions to `pip install SpeechRecognition pyaudio`

## Notes
- Telemetry P1 is intentionally minimal: `/health`, `/status`, `/` index page, and `/ws/events` broadcast. Orchestrator event hooks and basic UI controls are planned next.
- Security: Keep the server on trusted networks; basic auth is reserved in config and will be enforced in a future update.

## Upgrade guidance
- No breaking API changes.
- If you previously relied on running `packmind/orchestrator.py` directly, it now works without extra `PYTHONPATH` tweaks.
- For simulation-only examples/tests, prefer `from pidog_sim import Pidog`.
